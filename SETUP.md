# Настройка приложения «Рецепты» — пошагово для чайников

Это руководство проведёт тебя от пустого компьютера до работающего приложения.
Делай строго по порядку. Всё бесплатно.

Что мы настроим:
1. Node.js (среда для запуска)
2. Зависимости проекта
3. Ключ **Gemini** (генерация рецептов)
4. Проект **Supabase** (база данных + вход + хранилище фото)
5. Три SQL-скрипта в Supabase (таблица рецептов + бакет для фото + категории)
6. Настройку входа по ссылке (magic-link)
7. Файл `.env.local` с ключами
8. Запуск и проверку

> ⚠️ Секретные ключи (всё из `.env.local`) **никому не пересылай и не коммить в git**.
> Файл `.env.local` уже в `.gitignore`, так что случайно не попадёт в репозиторий.

---

## Шаг 1. Установи Node.js

Нужен Node.js версии 20 или новее.

1. Открой https://nodejs.org
2. Скачай версию **LTS** (большая зелёная кнопка), установи как обычную программу.
3. Проверь в терминале:
   ```bash
   node -v
   ```
   Должно вывести номер версии (например `v20.11.0` или выше). Если команда не
   найдена — перезапусти терминал.

---

## Шаг 2. Установи зависимости проекта

В терминале перейди в папку проекта и выполни:

```bash
cd путь/к/receitas
npm install
```

Это скачает все библиотеки (может занять минуту). Готово, когда команда
завершится без красных ошибок.

---

## Шаг 3. Получи ключ Gemini

1. Открой https://aistudio.google.com и войди Google-аккаунтом.
2. Слева (или по кнопке **Get API key** / **Создать API-ключ**) открой раздел
   **API keys**.
3. Нажми **Create API key** → выбери проект (или создай новый) → ключ появится.
4. Нажми **Copy** и сохрани его временно в блокнот. Он выглядит как длинная
   строка букв и цифр.

> Бесплатный тариф модели `gemini-3.1-flash-lite` — **500 запросов в сутки**.
> Для личного использования с запасом. Платить не нужно.

---

## Шаг 4. Создай проект Supabase

Supabase — это база данных, вход по email и хранилище фото, всё в одном.

1. Открой https://supabase.com → **Start your project** → войди (можно через GitHub).
2. Нажми **New project**.
3. Заполни:
   - **Name** — любое, например `receitas`
   - **Database Password** — придумай пароль (запиши, но для приложения он не
     понадобится) 
   - **Region** — ближайший к тебе регион
4. Нажми **Create new project** и подожди 1–2 минуты, пока проект создастся.

---

## Шаг 5. Возьми URL и ключ Supabase

Тебе нужны **два** значения.

### 5a. Project URL

1. В проекте слева внизу нажми шестерёнку **Settings**.
2. Открой **Data API**.
3. Скопируй **Project URL** — вид `https://xxxxxxxx.supabase.co`.

> ⚠️ **Важно:** копируй **только** адрес до `.supabase.co`.
> НЕ добавляй в конце `/rest/v1/` или любой другой хвост — иначе приложение не
> подключится (мы на этом уже спотыкались).

### 5b. Publishable key

1. Там же в **Settings** открой **API Keys**.
2. Вкладка **Publishable and secret API keys**.
3. В блоке **Publishable key** нажми **Copy** — ключ вида `sb_publishable_...`.

> **Secret key** (`sb_secret_...`) брать **не нужно** — приложение работает без него.
> Никогда его не копируй в чат и не коммить.

Сохрани оба значения в блокнот рядом с ключом Gemini.

---

## Шаг 6. Запусти SQL-скрипты в Supabase

В проекте лежат три файла с готовыми скриптами. Их нужно выполнить один раз.

1. В Supabase слева нажми **SQL Editor**.
2. Нажми **+ New query** (или «New snippet»).
3. Открой в проекте файл **`supabase/schema.sql`**, скопируй **всё** его
   содержимое, вставь в редактор и нажми **Run** (или Cmd/Ctrl + Enter).
   → Это создаёт таблицу рецептов и правила доступа. Должно написать `Success`.
4. Снова **+ New query**, открой файл **`supabase/storage.sql`**, скопируй всё,
   вставь, нажми **Run**.
   → Это создаёт хранилище для фото (бакет `recipe-photos`) и правила доступа.
5. Снова **+ New query**, открой файл **`supabase/categories.sql`**, скопируй всё,
   вставь, нажми **Run**.
   → Это создаёт категории рецептов и сразу засевает 4 дефолтных (Супы, Салаты,
   Основные блюда, Десерты). Их потом можно удалять и добавлять свои.

Если все три выполнились без красных ошибок — база готова.

---

## Шаг 7. Разреши вход по ссылке (magic-link)

Чтобы ссылка из письма возвращала тебя в приложение:

1. В Supabase слева открой **Authentication** → **URL Configuration**.
2. **Site URL** — впиши `http://localhost:3000`
3. **Redirect URLs** — нажми **Add URL** и впиши `http://localhost:3000/**`
   (двойная звёздочка в конце обязательна).
4. Сохрани (**Save**).

---

## Шаг 8. Создай файл `.env.local`

В корне проекта уже есть образец — **`.env.local.example`**. Сделай на его
основе рабочий файл:

1. Скопируй `.env.local.example` и назови копию **`.env.local`**
   (в терминале: `cp .env.local.example .env.local`).
2. Открой `.env.local` в редакторе и подставь свои значения:

```env
# из Шага 3
GEMINI_API_KEY=вставь_сюда_ключ_gemini
GEMINI_MODEL=gemini-3.1-flash-lite

# из Шага 5a — без /rest/v1/ на конце!
NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxx.supabase.co

# из Шага 5b
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=sb_publishable_...
```

3. Сохрани файл.

Куда что вставлять — короткая шпаргалка:

| Строка в `.env.local` | Что вставить | Откуда (шаг) |
|---|---|---|
| `GEMINI_API_KEY` | ключ Gemini | Шаг 3 (aistudio.google.com) |
| `GEMINI_MODEL` | `gemini-3.1-flash-lite` | оставь как есть |
| `NEXT_PUBLIC_SUPABASE_URL` | `https://…​.supabase.co` | Шаг 5a (Settings → Data API) |
| `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` | `sb_publishable_…` | Шаг 5b (Settings → API Keys) |

---

## Шаг 9. Проверь ключи (необязательно, но полезно)

Быстрый тест, что ключи рабочие (значения ключей на экран не выводятся):

```bash
npm run smoke
```

Ожидаемо: `Gemini: OK` и `Supabase: OK`. Если что-то `FAIL` — смотри раздел
«Частые проблемы» ниже.

---

## Шаг 10. Запусти приложение

```bash
npm run dev
```

Когда увидишь строку про `http://localhost:3000` — открой этот адрес в браузере.

Первый вход:
1. Введи свой email → **Прислать ссылку для входа**.
2. Открой письмо от Supabase (проверь папку «Спам») → перейди по ссылке.
3. Тебя вернёт в приложение — готово.
4. **+ Новый** → напиши запрос (например «борщ с говядиной») → **Сгенерировать**
   → выбери рецепт(ы) → **Сохранить**.
5. Открой рецепт: множитель порций, редактируемые количества, добавление фото.

Чтобы остановить приложение — в терминале нажми `Ctrl + C`.

## Частые проблемы

**«Модель вернула невалидный JSON» или рандомный рецепт по ссылке**
— норм для отдельных запросов, повтори. Если постоянно — проверь `GEMINI_MODEL`.

**`npm run smoke` → Supabase FAIL, статус 404**
— в `NEXT_PUBLIC_SUPABASE_URL` затесался хвост `/rest/v1/`. Убери его, оставь
только `https://…​.supabase.co`.

**`npm run smoke` → Supabase FAIL, статус 401**
— неверный или обрезанный `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`. Скопируй заново
из Settings → API Keys (именно **Publishable**, не Secret).

**Gemini FAIL, «quota» / 429**
— исчерпан дневной лимит (500/сутки у flash-lite). Подожди до сброса (сутки).

**Вход: ссылка из письма ведёт «в никуда» / ошибка**
— не выполнен Шаг 7. Проверь, что в Redirect URLs есть `http://localhost:3000/**`.

**Загрузка фото падает с ошибкой про RLS / bucket**
— не выполнен второй SQL-скрипт. Выполни `supabase/storage.sql` (Шаг 6).

**Категории не показываются**
— не выполнен третий SQL-скрипт. Выполни `supabase/categories.sql` (Шаг 6).
Приложение при этом не ломается — просто без категорий.

**«Пока пусто» после сохранения рецепта**
— не выполнен первый SQL-скрипт. Выполни `supabase/schema.sql` (Шаг 6).

## (Рекомендуется) Вход через Google

Встроенная почта Supabase для входа по ссылке жёстко лимитирована на бесплатном
тарифе (несколько писем в час — можно упереться в «email rate limit exceeded»).
Вход через Google это обходит: писем нет, вход в один клик. Email-вход при этом
остаётся запасным — ничего удалять не нужно.

Идея: получить в Google **Client ID** и **Client Secret** и вставить их в Supabase.
Кнопка «Войти через Google» в приложении уже есть — она заработает, как только
провайдер будет настроен (и локально, и на проде; ключи Google живут в Supabase,
не в коде).

### Часть A. Google Cloud Console (получить Client ID + Secret)

1. Открой https://console.cloud.google.com, войди Google-аккаунтом.
2. Вверху создай проект (выпадашка проектов → **New Project**), назови например
   `receitas`, выбери его.
3. Слева **APIs & Services → OAuth consent screen** (экран согласия):
   - User Type: **External** → Create
   - App name: `Рецепты`; support email и developer email — твой email → далее
   - Scopes — ничего не добавляй → **Save and continue**
   - **Test users → Add users** → впиши **свой** Google-email (чтобы вход работал,
     пока приложение в режиме Testing) → Save
   - Оставь режим **Testing** — для личного использования публиковать не нужно.
4. Слева **APIs & Services → Credentials**:
   - **Create Credentials → OAuth client ID**
   - Application type: **Web application**
   - **Authorized redirect URIs → Add URI** — сюда нужен callback-адрес Supabase
     (см. следующий пункт). Формат:
     `https://<твой-project-ref>.supabase.co/auth/v1/callback`
   - **Create** → скопируй **Client ID** и **Client Secret**.

> Точный callback-адрес не угадывай: открой Supabase → **Authentication →
> Providers → Google** — там показан **Callback URL (for OAuth)**. Скопируй его
> оттуда и вставь в Google (пункт 4). Так не ошибёшься с `project-ref`.

### Часть B. Supabase (включить провайдер Google)

1. Supabase → **Authentication → Providers** → **Google**.
2. Включи тумблер **Enable**.
3. Вставь **Client ID** и **Client Secret** из Части A.
4. **Save**.

### Проверка

Открой приложение (локально `npm run dev` или боевой адрес) → **Войти через
Google** → выбери аккаунт → попадаешь внутрь. Без писем.

### Если не работает

**Google выдаёт `redirect_uri_mismatch`**
— адрес в *Authorized redirect URIs* (Часть A, пункт 4) не совпадает с тем, что
показывает Supabase (Часть B). Скопируй callback из Supabase заново, вставь в
Google **ровно** как есть.

**`Unsupported provider` / кнопка ничего не делает**
— провайдер Google не включён или не сохранён в Supabase (Часть B).

## Публикация в интернет (деплой на Vercel)

Когда локально всё работает — можно выложить приложение в интернет, чтобы
пользоваться им с телефона по обычной ссылке. Хостинг **Vercel** — бесплатный.

Предварительно убедись, что локальная сборка проходит без ошибок:

```bash
npm run build
```

### Шаг Д1. Сохрани код в git

```bash
git add -A
git commit -m "готово к деплою"
```

> `.env.local` в git не попадёт (он в `.gitignore`) — секреты остаются только у
> тебя на компьютере и в настройках Vercel.

### Шаг Д2. Войди в Vercel

```bash
npx vercel login
```

Откроется браузер — выбери способ входа (можно Google/GitHub) и подтверди.

### Шаг Д3. Создай проект на Vercel

```bash
npx vercel link --yes
```

Команда создаст проект (по имени папки) и свяжет его с этой папкой.

### Шаг Д4. Загрузи переменные окружения на Vercel

Те же ключи, что в `.env.local`, нужно положить в настройки проекта Vercel
(на сервере своего `.env.local` нет). Выполни по одной команде на переменную —
каждая спросит **значение** (вставь из `.env.local`) и **окружение** (выбери
**Production**):

```bash
npx vercel env add GEMINI_API_KEY production
npx vercel env add GEMINI_MODEL production
npx vercel env add NEXT_PUBLIC_SUPABASE_URL production
npx vercel env add NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY production
```

Проверить, что все на месте (значения скрыты):

```bash
npx vercel env ls production
```

> ⚠️ Переменные `NEXT_PUBLIC_*` вшиваются в сборку, поэтому их нужно добавить
> **до** деплоя (следующий шаг). Если добавил после — просто повтори деплой.

### Шаг Д5. Задеплой в продакшн

```bash
npx vercel --prod
```

Vercel соберёт приложение на своих серверах и в конце напечатает **боевой адрес**
вида `https://твой-проект.vercel.app`. Запиши его.

### Шаг Д6. Разреши вход на боевом адресе (Supabase)

Как и для localhost (Шаг 7), нужно разрешить возврат по magic-link:

1. Supabase → **Authentication → URL Configuration**
2. **Site URL** → `https://твой-проект.vercel.app`
3. **Redirect URLs** → **Add URL** → `https://твой-проект.vercel.app/**`
   (строку `http://localhost:3000/**` оставь — чтобы локальная разработка работала)
4. **Save**

Без этого письмо со ссылкой на проде будет вести в никуда.

### Готово

Открой боевой адрес, войди по email и пользуйся. Приложение можно «установить»
на телефон: в браузере на телефоне открой сайт → меню → «Добавить на главный
экран».

### Как обновлять приложение после правок кода

```bash
git add -A && git commit -m "описание правок"
npx vercel --prod
```

> Переменные окружения заново вводить не нужно — они уже сохранены в проекте.

**Авто-деплой (по желанию).** Если подключить репозиторий к GitHub и импортировать
его в Vercel, каждый `git push` будет деплоить автоматически, без ручного
`vercel --prod`. Это отдельная настройка — не обязательна.

### Частые проблемы при деплое

**Сборка падает на Vercel, а локально `npm run build` проходит**
— чаще всего забыл добавить переменную окружения (Шаг Д4). Проверь
`npx vercel env ls production` и повтори деплой.

**На проде вход по ссылке не возвращает в приложение**
— не выполнен Шаг Д6 (нет боевого адреса в Redirect URLs Supabase).

**Страница просит войти в Vercel / показывает 401**
— это защита превью-деплоев. Пользуйся основным адресом проекта
(`https://твой-проект.vercel.app`), а не длинным адресом конкретного деплоя.
